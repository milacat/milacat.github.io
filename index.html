CountDownLath

发表于 2019-03-27 | 分类于 java并发变成基础

在遇到多线程的操作时，有时我们会遇到一个线程要等其他线程准备就绪或者等其他线程都执行完毕后执行，这时候CountDownLath便能派上用场，操作简单，十分好用。
CountDownLath的内结构如下：

Sync是 CountDownLath的一个内部类，继承自 AbstractQueuedSynchronizer，并重写了tryAcquireShared和tryReleaseShared方法。

private static final class Sync extends AbstractQueuedSynchronizer {
    private static final long serialVersionUID = 4982264981922014374L;

    Sync(int count) {
        setState(count);
    }

    int getCount() {
        return getState();
    }

    protected int tryAcquireShared(int acquires) {
        return (getState() == 0) ? 1 : -1;
    }

    protected boolean tryReleaseShared(int releases) {
        // Decrement count; signal when transition to zero
        for (;;) {
            int c = getState();
            if (c == 0)
                return false;
            int nextc = c-1;
            if (compareAndSetState(c, nextc))
                return nextc == 0;
        }
    }
}

CountDownLatch的初始化，n为需要观察的线程数，用来设置内部类Sync的state字段值
private static CountDownLatch latch = new CountDownLatch(n);
每个线程完成操作后，可以调用countDown方法，通过调用AQS的releaseShared方法使state的值减1.
下面是一个简单的示例：

public class CountDownLatchTest {

static CountDownLatch start = new CountDownLatch(1);
static CountDownLatch end;

public static void main(String[] args) throws InterruptedException {
    int threadCount = 10;
    end = new CountDownLatch(threadCount);
    for (int i = 0; i < threadCount; i++) {
        Thread thread = new Thread(new Runner(i));
        thread.start();
    }

    System.out.println("线程开始执行");
    start.countDown();
    end.await();
    System.out.println("所有线程执行完毕");
}

private static class Runner implements Runnable {

    private int i;

    public Runner(int i) {
        this.i = i;
    }

    public void run() {

        try {
            start.await();
        } catch (Exception e) {
        }
        System.out.println("线程" + i + "开始执行------");
        end.countDown();
    }
}

}
