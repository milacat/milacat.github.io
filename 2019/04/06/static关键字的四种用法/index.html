static关键字的四种用法

发表于 2019-04-06 | 分类于 java基础 | 阅读量 次

在日常开发或者看开发相关书籍时，经常会遇到static关键字。而我仅知道被static修饰的成员，能在类被初始化的时候加载，可以通过类名直接调用。但static关键字修饰的位置不同时，具体有哪些特性，总是了解后又很快忘记，所以整理到文章中，随时可做参考。

static修饰成员变量

所有程序编译运行的过程中，变量都是由系统自动分配内存进行存储的。static所修饰的变量(也就是静态变量，也叫做类变量)，类在被加载的同时，会被存储在静态区，被堆中的对象所共享，一直伴随程序运行，直到程序结束终止，存储空间才会被释放。普通变量（成员变量），是在对象被创建时才初始化的。

static修饰成员方法

static修饰的方法，便捷之处时候可以通过 类名.方法名 直接调用，在项目开发中，常被用到工具类中。被修饰的方法不属于任何某个对象，而是属于它所在的类，不需要消耗资源反复创建对象。
但是切不要滥用static修饰方法，因为被修饰后的方法在类第一次加载的时候，就已经在内存中了，直到程序结束后，该内存才会释放。所以比较占内存，而且在某些常些高并发常见下，会有线程安全问题。
补充概念：static方法是没有this的方法，this代表当前对象。而static方法不需要对象

static单独作为静态方法块

前面讲到过，sttaic修饰的成员会在类第一次被加载的时候被初始化，且在整个程序的运行过程中，只会初始化一次，不会进行多次初始化。 

    public class Cat extends Animal {

        static {
            System.out.println("cat static");
         }

    public Cat() {
        System.out.println("cat constructor");
    }

    public static void main(String[] args) {
        Cat cat = new Cat();
        System.out.println("-----------");
        Cat cat1 = new Cat();  //不会加载静态块，注意打印结果
    }
}

class Animal {
    //静态块会在第一次加载类时最先被初始化，且在程序的整个运行过程中，只会被初始化一次
    static {
        System.out.println("animal static");
    }

    public Animal() {
        System.out.println("animal constructor");
    }
}

static修饰类

普通类是不允许被static关键字修饰的，只有内部类可以。被static修饰的内部类可以直接作为一个普通类来使用，而不需实例一个外部类(非静态内部类只能通过外部类被实例化外部类对象.new 内部类（)。
